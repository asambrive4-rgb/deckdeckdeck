# DeckDeckDeck 통합 상단바 UI 배치 지시서

## 목적

DeckDeckDeck 앱의 상단 영역을 Windows 기본 제목 표시줄처럼 분리되어 보이지 않게 만들고, 앱 UI와 자연스럽게 통합된 **얇은 상단바**처럼 보이게 한다.

현재 화면에서 빨간색으로 표시된 최상단 영역을 별도의 기본 창 제목 영역처럼 남겨두지 않고, 앱 자체 UI의 일부처럼 사용한다.

---

## 핵심 방향

- 상단의 `DeckDeckDeck` 텍스트는 제거한다.
- 앱 내부에 크게 표시되는 `DeckDeckDeck` 제목도 제거한다.
- 기존의 큰 헤더 영역을 줄이고, 상단을 얇은 통합 바처럼 사용한다.
- 하단 상태 메시지 영역은 제거하고, 상태 메시지는 상단바 우측에 작게 배치한다.
- 설정 버튼은 본문 내부가 아니라 상단바 우측으로 이동한다.
- 뒤로 버튼은 필요한 화면에서 상단바 좌측에 배치한다.
- 텐키 카드 영역이 최대한 위로 올라오도록 한다.
- 전체적으로 예시 프로그램처럼 기본 창 프레임과 앱 UI가 따로 놀지 않는 느낌을 목표로 한다.

---

## 현재 문제

현재 앱은 다음처럼 보인다.

```text
[Windows 기본 제목 표시줄: 아이콘 + DeckDeckDeck + 최소화/최대화/닫기]
[앱 내부 여백]
[큰 제목: DeckDeckDeck]
[부제: 카테고리]
[설정 버튼]
[텐키 카드 영역]
[하단 상태 메시지]
```

이 구조의 문제는 다음과 같다.

- 제목 표시줄과 앱 내부 헤더가 분리되어 보여 상단 공간이 이중으로 낭비된다.
- `DeckDeckDeck` 텍스트가 제목 표시줄과 본문 안에 중복되어 시각적으로 무겁다.
- 설정 버튼이 본문 헤더 안에 있어 상단바 액션처럼 느껴지지 않는다.
- 하단 상태 메시지가 짧은 정보에 비해 너무 큰 공간을 차지한다.
- 텐키 카드가 화면의 핵심인데, 상단/하단 부가 영역 때문에 사용 가능한 공간이 줄어든다.

---

## 목표 구조

수정 후에는 다음 느낌이 되어야 한다.

```text
[통합 상단바: 뒤로 / 현재 위치 / 상태 메시지 / 설정 / 창 버튼]
[텐키 카드 또는 현재 화면 본문]
```

즉, 아래처럼 보이면 안 된다.

```text
제목 표시줄
앱 제목
본문
```

대신 아래처럼 보여야 한다.

```text
통합 상단바
본문
```

---

## 상단바 크기 기준

- 현재 Windows 기본 제목 표시줄은 스크린샷 기준 약 `32px ~ 36px` 정도로 보인다.
- 새 통합 상단바는 기존 기본 제목 표시줄보다 약간 더 넓게 잡는다.
- 권장 상단바 높이는 `48px`이다.
- 허용 범위는 `44px ~ 52px`이다.
- `40px` 이하는 너무 Windows 기본 제목 표시줄처럼 보일 수 있으므로 피한다.
- `56px` 이상은 다시 큰 헤더처럼 느껴질 수 있으므로 피한다.
- 상단바는 창 전체 너비를 가득 채운다.
- 상단바 내부 요소는 세로 중앙 정렬한다.
- 버튼 클릭 영역은 너무 작아지지 않게 하되, 본문보다 시각적으로 무겁게 보이면 안 된다.

### 권장 세부 크기

| 요소          |              권장 크기 |
| ----------- | -----------------: |
| 상단바 높이      |             `46px` |
| 상단바 허용 높이   |      `44px ~ 52px` |
| 뒤로 버튼 폭     |      `32px ~ 40px` |
| 설정 버튼 폭     |      `52px ~ 64px` |
| 상태 메시지 최대 폭 |          약 `160px` |
| 상태 메시지 처리   |            넘치면 말줄임 |
| 창 제어 버튼 영역  | 기본 창 버튼과 유사한 크기 유지 |

## 상단바 배경 기준

- 상단바 배경은 앱 배경과 통합되어 보여야 한다.
- 권장 배경색은 앱 메인 배경 계열인 `#FFF7D1`이다.
- 대안 배경색은 본문 표면 계열인 `#FFFDF4`이다.
- 상단바와 본문 사이에는 강한 구분선을 두지 않는다.
- 필요할 경우에만 아주 약한 구분선 `#E6CFA8`을 사용한다.
- 상단바가 Windows 기본 제목 표시줄처럼 흰색/회색 막대로 보이면 안 된다.
- 상단바와 본문이 서로 다른 앱 영역처럼 분리되어 보이면 안 된다.
- 상단바는 앱의 같은 표면 위에 놓인 기능 영역처럼 보여야 한다.

### 권장 배경 조합

| 항목 | 권장값 |
|---|---|
| 상단바 배경 1순위 | `#FFF7D1` |
| 상단바 배경 2순위 | `#FFFDF4` |
| 본문 패널 배경 | 기존 앱 표면색 유지 |
| 약한 구분선 | `#E6CFA8` |
| 강한 구분선 | 사용하지 않음 |
| 그림자 | 사용하지 않거나 매우 약하게 |

## 공통 상단바 배치

### 기본 형태

```text
┌─────────────────────────────────────────────────────────┐
│  ←     현재 위치                         상태   설정  ─ □ × │
├─────────────────────────────────────────────────────────┤
│                                                         │
│                      본문 영역                          │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 상단바 구성 요소

| 위치 | 요소 | 설명 |
|---|---|---|
| 좌측 | 뒤로 버튼 | 홈에서는 숨김. 카테고리, 설정, 편집 화면에서 표시 |
| 좌측 또는 중앙 | 현재 위치 | 현재 화면명 또는 카테고리명. 크게 강조하지 않음 |
| 우측 | 상태 메시지 | 기존 하단 메시지를 작게 표시 |
| 우측 | 설정 버튼 | 홈/카테고리 화면에서 표시 |
| 최우측 | 창 버튼 | 최소화, 최대화, 닫기 버튼을 앱 UI와 자연스럽게 유지 |

## 화면별 UI 배치

## 1. 홈 화면

### 배치

```text
┌─────────────────────────────────────────────────────────┐
│                                  준비됨   설정   ─ □ ×    │
├─────────────────────────────────────────────────────────┤
│                                                         │
│                    텐키 카드 영역                       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 처리

- 뒤로 버튼은 표시하지 않는다.
- 상단의 `DeckDeckDeck` 텍스트는 표시하지 않는다.
- 본문 내부의 큰 `DeckDeckDeck` 제목도 제거한다.
- `카테고리` 부제는 삭제하거나, 꼭 필요하다면 매우 작게만 표시한다.
- 설정 버튼은 상단바 우측에 둔다.
- 상태 메시지는 설정 버튼 왼쪽에 작게 둔다.
- 텐키 카드 영역이 기존보다 위로 올라와야 한다.

---

## 2. 카테고리 화면

### 배치

```text
┌─────────────────────────────────────────────────────────┐
│  ←   기본                              기본 카테고리  설정  ─ □ × │
├─────────────────────────────────────────────────────────┤
│                                                         │
│                    텐키 카드 영역                       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 처리

- 뒤로 버튼은 상단바 좌측에 둔다.
- 현재 카테고리명을 상단바에 작게 표시한다.
- 기존 본문 내부의 큰 제목/부제 블록은 제거한다.
- 설정 버튼은 상단바 우측에 둔다.
- 상태 메시지는 상단바 우측에 작게 표시한다.
- 텐키 카드 영역이 바로 본문 핵심으로 보이게 한다.

---

## 3. 설정 화면

### 배치

```text
┌─────────────────────────────────────────────────────────┐
│  ←   설정                              준비됨     ─ □ ×    │
├─────────────────────────────────────────────────────────┤
│                                                         │
│                    설정 내용                            │
│                                                         │
│                                      뒤로    저장        │
└─────────────────────────────────────────────────────────┘
```

### 처리

- 상단바 좌측에 뒤로 버튼을 둔다.
- 제목은 `설정` 정도로 작게 표시한다.
- 설정 화면 안에서는 설정 버튼을 다시 표시하지 않는다.
- 기존 큰 제목 헤더는 제거하거나 상단바로 흡수한다.
- 저장 버튼은 하단에 유지한다.
- 설정 내용은 상단바 바로 아래에서 시작한다.

- 삭제, 취소, 저장 버튼은 하단에 유지한다.

## 상태 메시지 처리

### 현재

```text
[본문]
[하단 상태 메시지 패널: 홈 / 준비됨 / 저장됨 등]
```

### 변경 후

```text
[상단바 우측: 준비됨]
[본문]
```

### 표시 원칙

- 상태 메시지는 상단바 우측의 작은 텍스트 또는 pill 형태로 표시한다.
- 너무 긴 메시지는 한 줄 안에서 잘리거나 생략되어도 된다.
- 상태 메시지가 본문보다 눈에 띄면 안 된다.
- 상태 메시지는 보조 정보이며, 텐키 카드나 주요 편집 내용보다 시각 우선순위가 낮아야 한다.

## 설정 버튼 처리

### 현재

- 홈/카테고리 화면의 본문 헤더 오른쪽에 설정 버튼이 있다.

### 변경 후

- 설정 버튼은 통합 상단바 우측에 둔다.
- 홈 화면과 카테고리 화면에서는 표시한다.
- 설정 화면과 편집 화면에서는 표시하지 않는 방향이 자연스럽다.
- 버튼은 큰 사각형 버튼보다 작고 가벼운 상단바 액션처럼 보여야 한다.

---

## 뒤로 버튼 처리

### 배치

- 상단바 좌측에 둔다.
- 홈 화면에서는 표시하지 않는다.
- 카테고리, 설정, 편집 화면에서는 표시한다.

### 시각 방향

- 큰 버튼보다 작고 가벼운 버튼 또는 아이콘형 버튼처럼 보이게 한다.
- 예시 프로그램처럼 상단바 안에 자연스럽게 들어간 느낌이어야 한다.
- 본문 내부의 별도 큰 뒤로 버튼처럼 보이지 않아야 한다.

---

## 제목 처리

### 제거할 제목

- 창 제목 영역의 `DeckDeckDeck` 텍스트
- 홈 화면 본문 내부의 큰 `DeckDeckDeck` 제목

### 유지할 정보

- 현재 화면이 어디인지 알려주는 짧은 위치 정보는 상단바에 작게 표시한다.

예시:

| 화면 | 상단바 위치 정보 |
|---|---|
| 홈 | 생략 가능 |
| 카테고리 | 카테고리명 |
| 설정 | 설정 |
| 카테고리 편집 | 카테고리 편집 / 슬롯 n |
| 실행 항목 편집 | 실행 항목 편집 / 슬롯 n |

## 여백 방향

- 상단과 본문 사이의 여백을 최소화한다.
- 기존의 큰 제목 영역이 사라진 만큼 텐키 카드가 위로 올라와야 한다.
- 앱 전체가 카드 안에 한 번 더 들어간 느낌을 줄인다.
- 상단바와 본문이 같은 앱 표면 위에 있는 것처럼 보여야 한다.
- 여백은 줄이되, 버튼 클릭 영역이 너무 작아지면 안 된다.

---

## 시각 스타일 방향

- 기존 DeckDeckDeck의 따뜻한 크림/브라운 계열 분위기는 유지한다.
- 상단바는 본문과 완전히 다른 색으로 튀지 않게 한다.
- 상단바는 Windows 기본 제목 표시줄처럼 보이면 안 된다.
- 상단바는 예시 프로그램처럼 앱의 자체 UI 일부로 보여야 한다.
- 설정, 뒤로, 상태 메시지는 모두 가볍고 작게 배치한다.
- 가장 먼저 눈에 들어와야 하는 것은 상단바가 아니라 본문 카드 영역이다.

## 최종 한 문장 요약

DeckDeckDeck의 상단 기본 창 제목 영역과 앱 내부 헤더를 하나의 얇은 통합 상단바처럼 합치고, `DeckDeckDeck` 중복 제목과 하단 상태 메시지 공간을 제거하여 텐키 카드 영역을 더 위로 끌어올린다. 새 상단바는 `48px` 높이를 기준으로 하고, 배경은 앱 메인 배경 계열인 `#FFF7D1`을 우선 사용한다.
