# 1. 제품 개요

## 1.1 한 줄 정의

> Windows에서 텐키 단축키와 텐키 모양 UI를 이용해, 저장해둔 프롬프트/문구를 빠르게 찾아 현재 커서 위치에 붙여넣는 개인용 데스크톱 앱.

---

## 1.2 제품의 핵심 아이디어

이 앱은 일반적인 목록형 스니펫 앱이 아니라, **키보드의 텐키 레이아웃 자체를 UI로 사용하는 프롬프트 런처**다.

사용자는 다음 방식으로 앱을 사용한다.

```text
Ctrl + Numpad 0
→ 홈 화면 진입
→ 텐키 키 입력
→ 덱 진입
→ 텐키 키 입력
→ 저장된 프롬프트 붙여넣기
```

또는:

```text
Ctrl + Numpad 숫자
→ 특정 덱 바로 진입
→ 텐키 키 입력
→ 저장된 프롬프트 붙여넣기
```

---

## 1.3 제품 방향

이 제품은 범용 런처, 매크로 도구, 팀 공유 도구가 아니다.

MVP는 아래 경험에 집중한다.

1. Windows 어디서든 텐키 단축키로 앱을 빠르게 호출한다.
2. 실제 텐키 모양의 UI에서 덱과 문구를 위치로 기억한다.
3. 긴 Markdown 프롬프트, 코드블록, 반복 문구를 원문 그대로 저장한다.
4. 현재 커서 위치에 빠르게 붙여넣는다.
5. 기존 클립보드는 가능한 한 복원한다.
6. 모든 데이터는 로컬 PC에 저장한다.

# 2. 문제 정의

## 2.1 현재 문제

LLM을 자주 사용할 때 반복적으로 입력하는 프롬프트가 많다.

예를 들면:

- 글 구조화 요청
- 번역 프롬프트
- 코드 리뷰 프롬프트
- 문서 요약 프롬프트
- PRD 작성 프롬프트
- 특정 말투/형식 변환 프롬프트
- 반복적인 작업 지시문
- 긴 Markdown 템플릿

이런 프롬프트를 매번 직접 입력하거나, 메모장/노션/문서에서 찾아 복사하는 과정은 번거롭다.

---

## 2.2 기존 방식의 불편함

| 기존 방식 | 문제점 |
|---|---|
| 메모 앱에 저장 | 찾고 복사하는 과정이 느림 |
| 클립보드 히스토리 | 구조화가 어렵고 오래된 항목 찾기 불편 |
| 일반 스니펫 앱 | 키보드 중심의 빠른 선택 UX가 부족할 수 있음 |
| AutoHotkey 스크립트 | 관리가 어렵고 UI가 부족함 |
| 런처 앱 | LLM 프롬프트 저장/붙여넣기에 특화되어 있지 않음 |

## 4.2 사용자 니즈

- 자주 쓰는 프롬프트를 덱별로 정리하고 싶다.

- 텍스트 제목보다 이미지 아이콘으로 빠르게 알아보고 싶다.

- 앱이 붙여넣기 후 자동으로 사라졌으면 좋겠다.
- 필요할 때는 붙여넣기 후 창이 남아 있도록 설정할 수 있으면 좋겠다.
- 앱의 데이터가 내 PC 안에만 저장되면 좋겠다.

| 로깅 | 단순 파일 로그 또는 Microsoft.Extensions.Logging | 오류 추적용 로컬 로그 |

## 5.2 기술 선택 이유

### C# / .NET / WPF

이 제품은 Windows 전용이며, 핵심 기능이 전역 단축키, 현재 커서 위치 붙여넣기, 클립보드 복원, 창 표시/숨김처럼 Windows OS와 밀접하게 연결되어 있다.

따라서 Electron, Tauri, Flutter 같은 크로스플랫폼 스택보다 Windows 네이티브 제어가 쉬운 C# / .NET / WPF가 MVP에 적합하다.

### WPF

WPF는 다음 요구사항과 잘 맞는다.

- 실제 텐키 모양의 고정 그리드 UI 구현
- 이미지 썸네일 표시
- 폼 기반 추가/수정 화면
- 창 숨김/표시 제어
- 키보드 이벤트 처리
- MVVM 구조 적용

### Win32 API P/Invoke

WPF만으로는 전역 단축키나 입력 주입을 충분히 안정적으로 처리하기 어렵다.

따라서 아래 기능은 Win32 API를 직접 호출한다.

| 기능 | 사용 API |
|---|---|
| 전역 단축키 등록 | `RegisterHotKey` |
| 전역 단축키 해제 | `UnregisterHotKey` |
| 앱 창 메시지 후킹 | `HwndSource` / Win32 message hook |
| Ctrl+V 입력 시뮬레이션 | `SendInput` |
| 창 활성화 보조 | 필요 시 `SetForegroundWindow` 계열 검토 |

### SQLite

앱 데이터는 개인 PC에만 저장되며 서버가 필요 없다.

SQLite는 아래 데이터 저장에 적합하다.

- 덱
- 문구
- 설정
- 카드 활성화 상태
- 생성/수정 일시
- 이미지 경로
- 썸네일 경로

이미지 원본과 썸네일은 DB에 바이너리로 넣지 않고, 앱 내부 폴더에 파일로 저장한다.

# 6. 핵심 UX 컨셉

## 6.1 텐키 기반 2단계 구조

앱은 기본적으로 다음 2단계 구조를 가진다.

```text
홈 화면
└── 덱 카드
    └── 문구 카드
```

즉:

```text
Ctrl + Numpad 0
→ 홈 화면
→ 덱 선택
→ 문구 선택
→ 붙여넣기
```

---

## 6.2 홈 화면

홈 화면은 **덱 선택 화면**이다.

각 텐키 카드는 하나의 덱을 의미한다.

| 키 | 홈 화면에서의 의미 |
|---|---|
| Numpad 1 | 덱 1 |
| Numpad 2 | 덱 2 |
| Numpad 3 | 덱 3 |
| Numpad / | 덱 `/` |
| Numpad * | 덱 `*` |
| Numpad + | 덱 `+` |
| Numpad . | 덱 `.` |

---

## 6.3 덱 화면

덱 화면은 **문구 선택 화면**이다.

각 텐키 카드는 하나의 저장된 문구를 의미한다.

| 키 | 덱 화면에서의 의미 |
|---|---|
| Numpad 1 | 문구 1 |
| Numpad 2 | 문구 2 |
| Numpad 3 | 문구 3 |
| Numpad / | 문구 `/` |
| Numpad * | 문구 `*` |
| Numpad + | 문구 `+` |
| Numpad . | 문구 `.` |

## 7.2 카드 활성화/비활성화

모든 카드는 각 카드의 편집 화면에서 켜고 끌 수 있어야 한다.

MVP에서는 카드 활성화 상태를 홈 화면과 덱 화면에 동일하게 적용한다.

| 카드 편집 | 결과 |
|---|---|
| Numpad / 비활성화 | 홈 화면과 덱 화면에서 `/` 카드 사용 안 함 |
| Numpad + 비활성화 | 홈 화면과 덱 화면에서 `+` 카드 사용 안 함 |
| Numpad . 비활성화 | 홈 화면과 덱 화면에서 `.` 카드 사용 안 함 |

---

## 7.3 비활성화된 카드의 UI

비활성화된 카드는 MVP에서 디자인 가이드의 `Disabled` 상태 기준으로 표시한다.

이유는 실제 텐키 위치감을 유지할 수 있기 때문이다.

| 상태 | UI |
|---|---|
| 활성 + 비어 있음 | 디자인 가이드의 `Empty` 상태 기준으로 추가 가능 표시 |
| 활성 + 데이터 있음 | 제목/이미지 표시 |
| 비활성 | 디자인 가이드의 `Disabled` 상태 기준으로 표시, 선택 불가 |

# 8. 주요 사용자 흐름

## 8.1 홈 화면 진입

### 목적

전체 덱을 텐키 UI로 확인하고 선택한다.

### 흐름

```text
사용자가 Ctrl + Numpad 0 입력
→ 앱 홈 화면 표시
→ 텐키 모양의 덱 카드 표시
```

### 요구사항

- `Ctrl + Numpad 0`은 홈 화면 진입 단축키다.
- 홈 화면은 텐키 레이아웃을 모방한다.
- 각 카드에는 덱 이름, 이미지 썸네일, 번호/기호가 표시된다.
- 빈 카드는 디자인 가이드의 `Empty` 상태 기준으로 추가 가능하게 표시된다.
- 비활성화된 카드는 선택할 수 없다.

---

## 8.2 홈 화면에서 덱 진입

### 데이터가 있는 카드일 경우

```text
홈 화면 열린 상태
→ 사용자가 Numpad 3 입력
→ 3번 덱 화면으로 이동
```

### 빈 카드일 경우

```text

→ 사용자가 비어 있는 Numpad 3 입력
→ 새 덱 추가 화면 열림
```

### 마우스 사용

```text
홈 화면 열린 상태
→ 사용자가 빈 카드 클릭
→ 새 덱 추가 화면 열림
```

## 8.3 덱 바로 진입

### 흐름

```text
사용자가 Ctrl + Numpad 1 입력
→ 1번 덱 화면 바로 열림
```

### 요구사항

- `Ctrl + Numpad 1~9`는 해당 숫자 덱으로 바로 진입한다.
- 해당 카드에 덱가 없으면 새 덱 추가 화면을 열 수 있다.
- 해당 카드가 비활성화되어 있으면 아무 동작을 하지 않거나 안내를 표시한다.

### MVP 기본안

| 단축키 | 동작 |
|---|---|
| Ctrl + Numpad 0 | 홈 화면 |
| Ctrl + Numpad 1~9 | 숫자 덱 바로 진입 |
| Ctrl + Numpad 기호 | MVP에서는 직접 진입 미지원, 홈에서 접근 가능 |

## 8.4 문구 붙여넣기

### 흐름

```text
덱 화면 열린 상태
→ 사용자가 Numpad 5 입력
→ 5번 카드의 문구를 현재 커서 위치에 붙여넣기
→ 앱 창 자동 숨김
→ 기존 클립보드 복원
```

### 요구사항

- 문구는 현재 포커스가 있는 입력 위치에 붙여넣는다.
- 붙여넣기 후 창은 기본적으로 자동 숨김 처리한다.
- 자동 숨김 여부는 설정에서 변경 가능해야 한다.
- 기존 클립보드는 붙여넣기 후 복원한다.
- 줄바꿈, Markdown, 코드블록이 보존되어야 한다.
- 붙여넣기 모드에서는 앱 창이 가능한 한 원래 입력창의 포커스를 빼앗지 않는 비활성 선택 창처럼 동작해야 한다.
- 사용자가 문구 카드를 마우스로 클릭해도 원래 입력 커서 위치가 유지되어야 한다.
- 붙여넣기 모드에서 텐키 입력은 대상 앱에 숫자나 기호로 전달되지 않고, 앱의 카드 선택으로 처리되어야 한다.
- 빈 카드 선택, 카드 톱니모양 아이콘, 추가/수정 화면 진입처럼 텍스트 입력이 필요한 상황에서는 편집 모드로 전환하고 앱 창이 정상적으로 포커스를 가져야 한다.

## 14.2 텐키 전용 정책

- 상단 숫자키는 기본적으로 사용하지 않는다.
- 텐키 숫자와 텐키 기호만 카드 조작에 사용한다.
- NumLock 상태에 따른 동작 차이는 최소화해야 한다.
- NumLock이 꺼져 있어 동작이 제한될 경우 사용자에게 안내해야 한다.

## 14.3 붙여넣기 모드의 임시 텐키 캡처

붙여넣기 모드에서 앱이 화면에 떠 있는 동안에는 텐키 입력을 임시로 앱의 카드 선택 입력으로 처리한다.

| 상태 | 텐키 동작 |
|---|---|
| 앱이 숨겨져 있음 | 대상 앱의 일반 숫자/기호 입력으로 동작 |
| 붙여넣기 모드로 앱이 표시됨 | DeckDeckDeck의 카드 선택으로 동작 |
| 문구 붙여넣기 완료 | 임시 캡처 해제 후 일반 텐키 입력으로 복귀 |
| 편집 모드 진입 | 임시 캡처 해제, 앱이 정상 포커스를 가져감 |

임시 캡처는 사용자가 문구를 고르는 짧은 시간에만 동작해야 하며, 앱이 숨겨진 뒤에도 남아 있으면 안 된다.

# 15. 붙여넣기 정책

## 15.1 붙여넣기 모드와 편집 모드

앱은 붙여넣기 경험을 위해 두 가지 창 동작 모드를 구분한다.

| 모드 | 목적 | 포커스 정책 |
|---|---|---|
| 붙여넣기 모드 | 기존 문구를 빠르게 선택해 현재 입력 위치에 붙여넣기 | 앱 창은 가능한 한 비활성 선택 창처럼 표시되고, 원래 입력창 포커스를 유지 |
| 편집 모드 | 덱/문구 생성, 수정, 삭제 | 앱 창이 정상적으로 활성화되고 텍스트 입력 포커스를 가져감 |

붙여넣기 모드에서는 원래 입력하던 앱의 커서 위치를 보존하는 것이 우선이다.

편집 모드에서는 사용자가 앱 내부 입력 필드에 내용을 입력해야 하므로 앱이 포커스를 가져가는 것이 정상 동작이다.

---

## 15.2 기본 방식

문구 실행 시 앱은 다음 순서로 동작해야 한다.

```text
1. 앱 호출 전 또는 붙여넣기 모드 진입 전 대상 입력창 핸들 기억
2. 앱 창을 비활성 선택 창으로 표시하고 임시 텐키 캡처 활성화
3. 현재 클립보드 내용 백업
4. 선택한 문구 내용을 클립보드에 임시 저장
5. 대상 입력창 포커스를 유지하거나 복원한 뒤 Ctrl+V 입력 전송
6. 기존 클립보드 내용 복원
7. 설정에 따라 앱 창 숨김
8. 임시 텐키 캡처 해제
```

---

## 15.3 클립보드 복원

기본값은 **ON**이다.

| 상황 | 기대 동작 |
|---|---|
| 일반 텍스트 클립보드 | 원래 텍스트 복원 |
| 이미지 클립보드 | 가능하면 원래 이미지 복원 |
| 복합 형식 클립보드 | 가능한 범위에서 복원 |
| 복원 실패 | 앱 사용을 방해하지 않고 조용히 실패 처리 |

---

## 15.4 구현상 주의점

- 클립보드 백업은 단순 문자열이 아니라 가능한 경우 `IDataObject` 단위로 보관한다.
- 일부 앱은 붙여넣기 처리 시간이 느릴 수 있으므로, 클립보드 복원 전 기본 `500ms` 지연을 둔다.
- 관리자 권한으로 실행 중인 앱, 보안 프로그램, 일부 게임, 보호된 입력창에서는 `SendInput`이 동작하지 않을 수 있다.
- `SendInput`에 전달하는 Win32 `INPUT` 구조체는 `MOUSEINPUT`, `KEYBDINPUT`, `HARDWAREINPUT` union을 모두 포함해 네이티브 구조체 크기와 맞춰야 한다.
- 붙여넣기 실패 시 앱은 종료되지 않아야 한다.
- 붙여넣기 실패 상황은 로컬 로그에 남길 수 있다.
- 비활성 선택 창이 모든 Windows 앱에서 포커스 보존을 보장하지는 못하므로, 대상 입력창 핸들 저장과 `SetForegroundWindow` 기반 복원은 best-effort로 처리한다.
- 임시 텐키 캡처는 앱이 붙여넣기 모드일 때만 활성화하고, 편집 화면 진입/창 숨김/앱 종료 시 반드시 해제한다.

# 16. 관리자 권한 앱 대응

## 16.1 요구사항

일부 Windows 앱은 관리자 권한으로 실행될 수 있다.

이 경우 일반 권한 앱이 키 입력이나 붙여넣기를 전달하지 못할 수 있다.

따라서 앱은 다음을 지원해야 한다.

| 요구사항 | 설명 |
|---|---|
| 관리자 권한 실행 안내 | 관리자 앱에서도 사용하려면 앱도 관리자 권한이 필요할 수 있음을 안내 |
| 관리자 권한 실행 옵션 안내 | 사용자가 앱을 관리자 권한으로 실행할 수 있도록 안내 |
| 제한 사항 표시 | 보안 프로그램, 일부 게임, 보호된 입력창에서는 동작 보장 불가 |

MVP에서는 앱이 자동으로 권한 상승을 시도하지 않는다.

| 오류 복구 | 이미지 파일 누락, 클립보드 실패 등에도 앱이 종료되지 않아야 함 |
| 중복 입력 방지 | 단축키 연타 시 의도치 않은 중복 실행 방지 |
| 단일 실행 | 같은 앱 인스턴스가 여러 개 떠서 핫키 충돌이 나지 않아야 함 |
| DB 손상 대응 | DB 접근 실패 시 오류 메시지 또는 로그를 남기고 앱이 무한 크래시하지 않아야 함 |

# 25. PRD 요약

이 제품은 **Windows용 개인 프롬프트 런처**다.

핵심은 다음 다섯 가지다.

1. **텐키 UI**
   - 실제 텐키 레이아웃을 모방한다.
   - 숫자뿐 아니라 `/`, `*`, `-`, `+`, `.`까지 카드로 사용한다.

2. **2단계 선택 구조**
   - 홈 화면에서 덱 선택
   - 덱 화면에서 문구 선택

3. **LLM 프롬프트에 최적화**
   - 긴 텍스트
   - 줄바꿈
   - Markdown
   - 코드블록
   - 반복 사용 프롬프트 저장

4. **빠른 인지와 실행**
   - 각 카드에 이미지 첨부 가능
   - 앱이 자동으로 썸네일 처리
   - 텍스트보다 이미지와 위치 기억으로 빠르게 실행
   - 선택 즉시 붙여넣기
   - 기존 클립보드 복원
   - 붙여넣기 후 기본 자동 숨김

5. **Windows 네이티브 구현**
   - C# / .NET 10 LTS / WPF 기반 구현
   - Win32 API로 전역 단축키와 붙여넣기 처리
   - SQLite와 AppData 폴더로 로컬 데이터 저장
   - 배포, 서버, 계정, 클라우드 없이 개인용으로 사용

이 PRD의 MVP 범위는 **프롬프트 저장, 텐키 선택, 이미지 카드, 로컬 데이터 저장, Windows 붙여넣기 경험**에 집중한다.

키 리매핑, 프로그램 실행, 볼륨 조절, 매크로, 배포/설치 프로그램은 MVP 이후 또는 범위 밖으로 둔다.

# 26. 용어집

이 섹션은 사용자가 수정사항을 말할 때 쓰는 공통 단어를 정리한다.  
아래 용어를 기준으로 말하면, 요구사항과 구현 범위를 더 정확히 맞출 수 있다.

## 26.1 화면 용어

| 용어 | 뜻 | 이렇게 말하면 됨 |
|---|---|---|
| 홈 화면 | 앱을 열었을 때 처음 보이는 덱 선택 화면 | "홈 화면에서 1번 카드를 더 크게 보여줘" |
| 덱 화면 | 특정 덱 안의 문구 카드를 고르는 화면 | "덱 화면에서 뒤로가기 버튼을 추가해줘" |
| 덱 추가/수정 화면 | 홈 화면 카드에서 덱을 새로 만들거나 고치는 화면 | "덱 수정 화면에 이미지 삭제 버튼을 넣어줘" |
| 문구 추가/수정 화면 | 덱 안 카드에 붙여넣을 문구를 새로 만들거나 고치는 화면 | "문구 수정 화면에서 내용 입력칸을 더 크게 해줘" |
| 설정 화면 | 자동 숨김, 클립보드 복원 같은 앱 설정을 바꾸는 화면 | "설정 화면에 클립보드 복원 옵션을 보여줘" |

## 26.2 카드 용어

| 용어       | 뜻                                       | 이렇게 말하면 됨                    |
| -------- | --------------------------------------- | ---------------------------- |
| 카드       | 텐키의 한 키에 대응되는 사각형 자리                    | "카드 간격을 조금 줄여줘"              |
| 카드 키     | 카드에 대응되는 실제 텐키 키. 예: `1`, `2`, `/`, `+` | "`+` 카드를 비활성화할 수 있게 해줘"      |
| 홈 카드     | 홈 화면에서 덱을 담는 카드                      | "홈 카드에 덱 이미지가 잘 보이게 해줘"   |
| 덱 카드  | 홈 화면에서 하나의 덱을 의미하는 카드                | "덱 카드 이름이 길면 두 줄로 보여줘"    |
| 문구 카드    | 덱 화면에서 하나의 문구를 의미하는 카드               | "문구 카드를 누르면 바로 붙여넣기 되게 해줘"   |
| 빈 카드     | 아직 덱이나 문구가 들어 있지 않은 카드                | "빈 카드는 추가 가능하게 보여줘"          |
| 활성 카드    | 선택하거나 실행할 수 있는 카드                       | "활성 카드만 키보드로 선택되게 해줘"        |
| 비활성 카드   | 디자인 가이드의 `Disabled` 상태로 보이고 선택할 수 없는 카드 | "비활성 카드도 톱니 아이콘은 누를 수 있게 해줘" |
| 카드 편집    | 카드의 이름, 이미지, 활성화 상태 등을 바꾸는 기능           | "카드 편집 버튼 위치를 오른쪽 위로 옮겨줘"    |
| 톱니모양 아이콘 | 카드 편집 화면으로 들어가는 작은 설정 아이콘               | "톱니모양 아이콘이 너무 작아서 키워줘"       |

## 26.4 동작 용어

| 용어 | 뜻 | 이렇게 말하면 됨 |
|---|---|---|
| 홈 호출 | 단축키로 홈 화면을 여는 동작 | "`Ctrl + Numpad 0`으로 홈 호출되게 해줘" |
| 덱 바로 진입 | 홈 화면을 거치지 않고 특정 덱 화면을 여는 동작 | "`Ctrl + Numpad 3`이면 3번 덱으로 바로 가게 해줘" |
| 문구 붙여넣기 | 선택한 문구를 현재 커서 위치에 넣는 동작 | "문구 붙여넣기 후 창을 닫아줘" |
| 자동 숨김 | 문구 붙여넣기 후 앱 창을 자동으로 숨기는 동작 | "자동 숨김을 끌 수 있게 해줘" |
| 클립보드 복원 | 붙여넣기 전에 있던 클립보드 내용을 다시 돌려놓는 동작 | "클립보드 복원 실패 시 로그를 남겨줘" |
| 붙여넣기 모드 | 기존 입력 위치를 유지한 채 문구를 빠르게 고르는 상태 | "붙여넣기 모드에서는 앱이 포커스를 뺏지 않게 해줘" |
| 편집 모드 | 덱이나 문구를 입력하고 수정하는 상태 | "편집 모드에서는 입력칸에 바로 커서가 가게 해줘" |
| 임시 텐키 캡처 | 붙여넣기 모드에서 텐키 입력을 대상 앱이 아니라 DeckDeckDeck 카드 선택으로 처리하는 동작 | "창이 숨겨지면 임시 텐키 캡처를 반드시 해제해줘" |
| 포커스 | 현재 키보드 입력을 받고 있는 창이나 입력칸 | "문구 카드를 클릭해도 원래 입력창 포커스가 유지되게 해줘" |

## 26.5 단축키와 키 용어

| 용어 | 뜻 | 이렇게 말하면 됨 |
|---|---|---|
| 텐키 | 키보드 오른쪽 숫자 키패드 영역 | "텐키 모양 그대로 카드를 배치해줘" |
| Numpad | 텐키 키를 코드나 문서에서 부르는 이름 | "`Numpad +` 카드를 추가해줘" |

| 홈 단축키 | 홈 화면을 여는 단축키. MVP 기본값은 `Ctrl + Numpad 0` | "홈 단축키를 설정 화면에 표시해줘" |
| 직접 진입 단축키 | 특정 숫자 덱으로 바로 들어가는 단축키. MVP 기본값은 `Ctrl + Numpad 1~9` | "직접 진입 단축키는 숫자만 지원하게 해줘" |
