# 1. 제품 개요

## 1.1 한 줄 정의

> Windows에서 텐키 단축키와 텐키 모양 UI를 이용해, 저장해둔 프롬프트/문구를 빠르게 찾아 현재 커서 위치에 붙여넣는 개인용 데스크톱 앱.

---

## 1.2 제품의 핵심 아이디어

이 앱은 일반적인 목록형 스니펫 앱이 아니라, **키보드의 텐키 레이아웃 자체를 UI로 사용하는 프롬프트 런처**다.

사용자는 다음 방식으로 앱을 사용한다.

```text
Ctrl + Numpad 0
→ 홈 화면 진입
→ 텐키 키 입력
→ 카테고리 진입
→ 텐키 키 입력
→ 저장된 프롬프트 붙여넣기
```

또는:

```text
Ctrl + Numpad 숫자
→ 특정 카테고리 바로 진입
→ 텐키 키 입력
→ 저장된 프롬프트 붙여넣기
```

---

## 1.3 제품 방향

이 제품은 범용 런처, 매크로 도구, 팀 공유 도구가 아니다.

MVP는 아래 경험에 집중한다.

1. Windows 어디서든 텐키 단축키로 앱을 빠르게 호출한다.
2. 실제 텐키 모양의 UI에서 카테고리와 문구를 위치로 기억한다.
3. 긴 Markdown 프롬프트, 코드블록, 반복 문구를 원문 그대로 저장한다.
4. 현재 커서 위치에 빠르게 붙여넣는다.
5. 기존 클립보드는 가능한 한 복원한다.
6. 모든 데이터는 로컬 PC에 저장한다.

# 2. 문제 정의

## 2.1 현재 문제

LLM을 자주 사용할 때 반복적으로 입력하는 프롬프트가 많다.

예를 들면:

- 글 구조화 요청
- 번역 프롬프트
- 코드 리뷰 프롬프트
- 문서 요약 프롬프트
- PRD 작성 프롬프트
- 특정 말투/형식 변환 프롬프트
- 반복적인 작업 지시문
- 긴 Markdown 템플릿

이런 프롬프트를 매번 직접 입력하거나, 메모장/노션/문서에서 찾아 복사하는 과정은 번거롭다.

---

## 2.2 기존 방식의 불편함

| 기존 방식 | 문제점 |
|---|---|
| 메모 앱에 저장 | 찾고 복사하는 과정이 느림 |
| 클립보드 히스토리 | 구조화가 어렵고 오래된 항목 찾기 불편 |
| 일반 스니펫 앱 | 키보드 중심의 빠른 선택 UX가 부족할 수 있음 |
| AutoHotkey 스크립트 | 관리가 어렵고 UI가 부족함 |
| 런처 앱 | LLM 프롬프트 저장/붙여넣기에 특화되어 있지 않음 |

## 5.2 기술 선택 이유

### C# / .NET / WPF

이 제품은 Windows 전용이며, 핵심 기능이 전역 단축키, 현재 커서 위치 붙여넣기, 클립보드 복원, 창 표시/숨김처럼 Windows OS와 밀접하게 연결되어 있다.

따라서 Electron, Tauri, Flutter 같은 크로스플랫폼 스택보다 Windows 네이티브 제어가 쉬운 C# / .NET / WPF가 MVP에 적합하다.

### WPF

WPF는 다음 요구사항과 잘 맞는다.

- 실제 텐키 모양의 고정 그리드 UI 구현
- 이미지 썸네일 표시
- 폼 기반 추가/수정 화면
- 창 숨김/표시 제어
- 키보드 이벤트 처리
- MVVM 구조 적용

### Win32 API P/Invoke

WPF만으로는 전역 단축키나 입력 주입을 충분히 안정적으로 처리하기 어렵다.

따라서 아래 기능은 Win32 API를 직접 호출한다.

| 기능 | 사용 API |
|---|---|
| 전역 단축키 등록 | `RegisterHotKey` |
| 전역 단축키 해제 | `UnregisterHotKey` |
| 앱 창 메시지 후킹 | `HwndSource` / Win32 message hook |
| Ctrl+V 입력 시뮬레이션 | `SendInput` |
| 창 활성화 보조 | 필요 시 `SetForegroundWindow` 계열 검토 |

### SQLite

앱 데이터는 개인 PC에만 저장되며 서버가 필요 없다.

SQLite는 아래 데이터 저장에 적합하다.

- 카테고리
- 문구
- 설정
- 슬롯 활성화 상태
- 생성/수정 일시
- 이미지 경로
- 썸네일 경로

이미지 원본과 썸네일은 DB에 바이너리로 넣지 않고, 앱 내부 폴더에 파일로 저장한다.

# 6. 핵심 UX 컨셉

## 6.1 텐키 기반 2단계 구조

앱은 기본적으로 다음 2단계 구조를 가진다.

```text
홈 화면
└── 카테고리 슬롯
    └── 문구 슬롯
```

즉:

```text
Ctrl + Numpad 0
→ 홈 화면
→ 카테고리 선택
→ 문구 선택
→ 붙여넣기
```

---

## 6.2 홈 화면

홈 화면은 **카테고리 선택 화면**이다.

각 텐키 슬롯은 하나의 카테고리를 의미한다.

| 키 | 홈 화면에서의 의미 |
|---|---|
| Numpad 1 | 카테고리 1 |
| Numpad 2 | 카테고리 2 |
| Numpad 3 | 카테고리 3 |
| Numpad / | 카테고리 `/` |
| Numpad * | 카테고리 `*` |
| Numpad + | 카테고리 `+` |
| Numpad . | 카테고리 `.` |

---

## 6.3 카테고리 화면

카테고리 화면은 **문구 선택 화면**이다.

각 텐키 슬롯은 하나의 저장된 문구를 의미한다.

| 키 | 카테고리 화면에서의 의미 |
|---|---|
| Numpad 1 | 문구 1 |
| Numpad 2 | 문구 2 |
| Numpad 3 | 문구 3 |
| Numpad / | 문구 `/` |
| Numpad * | 문구 `*` |
| Numpad + | 문구 `+` |
| Numpad . | 문구 `.` |

## 7.2 슬롯 활성화/비활성화

모든 슬롯은 설정에서 켜고 끌 수 있어야 한다.

MVP에서는 슬롯 활성화 설정을 홈 화면과 카테고리 화면에 동일하게 적용한다.

| 설정 | 결과 |
|---|---|
| Numpad / 비활성화 | 홈 화면과 카테고리 화면에서 `/` 슬롯 사용 안 함 |
| Numpad + 비활성화 | 홈 화면과 카테고리 화면에서 `+` 슬롯 사용 안 함 |
| Numpad . 비활성화 | 홈 화면과 카테고리 화면에서 `.` 슬롯 사용 안 함 |

---

## 7.3 비활성화된 슬롯의 UI

비활성화된 슬롯은 MVP에서 **흐리게 표시**한다.

이유는 실제 텐키 위치감을 유지할 수 있기 때문이다.

| 상태 | UI |
|---|---|
| 활성 + 비어 있음 | `+` 아이콘 표시 |
| 활성 + 데이터 있음 | 제목/이미지 표시 |
| 비활성 | 흐리게 표시, 선택 불가 |

# 8. 주요 사용자 흐름

## 8.1 홈 화면 진입

### 목적

전체 카테고리를 텐키 UI로 확인하고 선택한다.

### 흐름

```text
사용자가 Ctrl + Numpad 0 입력
→ 앱 홈 화면 표시
→ 텐키 모양의 카테고리 슬롯 표시
```

### 요구사항

- `Ctrl + Numpad 0`은 홈 화면 진입 단축키다.
- 홈 화면은 텐키 레이아웃을 모방한다.
- 각 슬롯에는 카테고리 이름, 이미지 썸네일, 번호/기호가 표시된다.
- 빈 슬롯은 `+` 아이콘으로 표시된다.
- 비활성화된 슬롯은 선택할 수 없다.

---

## 8.2 홈 화면에서 카테고리 진입

### 데이터가 있는 슬롯일 경우

```text
홈 화면 열린 상태
→ 사용자가 Numpad 3 입력
→ 3번 카테고리 화면으로 이동
```

### 빈 슬롯일 경우

```text
홈 화면 열린 상태
→ 사용자가 비어 있는 Numpad 3 입력
```

```text
홈 화면 열린 상태
```

## 8.3 카테고리 바로 진입

### 흐름

```text
사용자가 Ctrl + Numpad 1 입력
→ 1번 카테고리 화면 바로 열림
```

### 요구사항

- `Ctrl + Numpad 1~9`는 해당 숫자 카테고리로 바로 진입한다.
- 해당 슬롯에 카테고리가 없으면 새 카테고리 추가 화면을 열 수 있다.
- 해당 슬롯이 비활성화되어 있으면 아무 동작을 하지 않거나 안내를 표시한다.

### MVP 기본안

| 단축키 | 동작 |
|---|---|
| Ctrl + Numpad 0 | 홈 화면 |
| Ctrl + Numpad 1~9 | 숫자 카테고리 바로 진입 |
| Ctrl + Numpad 기호 | MVP에서는 직접 진입 미지원, 홈에서 접근 가능 |

## 8.4 문구 붙여넣기

### 흐름

```text
카테고리 화면 열린 상태
→ 사용자가 Numpad 5 입력
→ 5번 슬롯의 문구를 현재 커서 위치에 붙여넣기
→ 앱 창 자동 숨김
→ 기존 클립보드 복원
```

### 요구사항

- 문구는 현재 포커스가 있는 입력 위치에 붙여넣는다.
- 붙여넣기 후 창은 기본적으로 자동 숨김 처리한다.
- 자동 숨김 여부는 설정에서 변경 가능해야 한다.
- 기존 클립보드는 붙여넣기 후 복원한다.
- 줄바꿈, Markdown, 코드블록이 보존되어야 한다.

## 12.2 이미지 저장 정책

이미지를 선택하면 원본 파일 경로만 참조하지 않고 앱 내부 폴더로 복사한다.

```text
images/originals/{guid}{extension}
images/thumbnails/{guid}.png
```

이유:

- 원본 이미지가 이동/삭제되어도 앱에서 계속 표시 가능
- 썸네일 표시 속도 개선
- 데이터 백업/복원이 쉬움

## 14.2 텐키 전용 정책

- 상단 숫자키는 기본적으로 사용하지 않는다.
- 텐키 숫자와 텐키 기호만 슬롯 조작에 사용한다.
- NumLock 상태에 따른 동작 차이는 최소화해야 한다.
- NumLock이 꺼져 있어 동작이 제한될 경우 사용자에게 안내해야 한다.

# 15. 붙여넣기 정책

## 15.1 기본 방식

문구 실행 시 앱은 다음 순서로 동작해야 한다.

```text
1. 현재 클립보드 내용 백업
2. 선택한 문구 내용을 클립보드에 임시 저장
3. 현재 포커스 위치에 Ctrl+V 입력 전송
4. 기존 클립보드 내용 복원
5. 설정에 따라 앱 창 숨김
```

---

## 15.2 클립보드 복원

기본값은 **ON**이다.

| 상황 | 기대 동작 |
|---|---|
| 일반 텍스트 클립보드 | 원래 텍스트 복원 |
| 이미지 클립보드 | 가능하면 원래 이미지 복원 |
| 복합 형식 클립보드 | 가능한 범위에서 복원 |
| 복원 실패 | 앱 사용을 방해하지 않고 조용히 실패 처리 또는 작은 알림 표시 |

---

## 15.3 구현상 주의점

- 클립보드 백업은 단순 문자열이 아니라 가능한 경우 `IDataObject` 단위로 보관한다.
- 일부 앱은 붙여넣기 처리 시간이 느릴 수 있으므로, 복원 타이밍에 짧은 지연이 필요할 수 있다.
- 관리자 권한으로 실행 중인 앱, 보안 프로그램, 일부 게임, 보호된 입력창에서는 `SendInput`이 동작하지 않을 수 있다.
- 붙여넣기 실패 시 앱은 종료되지 않아야 한다.
- 붙여넣기 실패 상황은 로컬 로그에 남길 수 있다.

# 16. 관리자 권한 앱 대응

## 16.1 요구사항

일부 Windows 앱은 관리자 권한으로 실행될 수 있다.

이 경우 일반 권한 앱이 키 입력이나 붙여넣기를 전달하지 못할 수 있다.

따라서 앱은 다음을 지원해야 한다.

| 요구사항 | 설명 |
|---|---|
| 관리자 권한 실행 안내 | 관리자 앱에서도 사용하려면 앱도 관리자 권한이 필요할 수 있음을 안내 |
| 관리자 권한 실행 옵션 안내 | 사용자가 앱을 관리자 권한으로 실행할 수 있도록 안내 |
| 제한 사항 표시 | 보안 프로그램, 일부 게임, 보호된 입력창에서는 동작 보장 불가 |

MVP에서는 앱이 자동으로 권한 상승을 시도하지 않는다.

# 21. 성공 기준

## 21.1 기능 성공 기준

| 기준 | 설명 |
|---|---|
| 홈 호출 성공 | Ctrl + Numpad 0으로 홈 화면이 열린다 |
| 카테고리 생성 성공 | 빈 슬롯에서 카테고리를 만들 수 있다 |
| 문구 생성 성공 | 카테고리 안 빈 슬롯에서 문구를 만들 수 있다 |
| 이미지 표시 성공 | 카테고리/문구에 이미지를 붙이고 썸네일로 볼 수 있다 |
| 텐키 선택 성공 | 마우스 없이 텐키 입력으로 탐색 가능하다 |
| 붙여넣기 성공 | 저장한 긴 Markdown 문구가 현재 위치에 붙여넣어진다 |
| 클립보드 복원 성공 | 붙여넣기 후 기존 클립보드가 유지된다 |
| 자동 숨김 성공 | 붙여넣기 후 창이 자동으로 숨겨진다 |
| 설정 유지 성공 | 앱 재시작 후 설정과 데이터가 유지된다 |
| 로컬 저장 성공 | DB, 이미지, 썸네일이 앱 데이터 폴더에 저장된다 |
| 권한 제한 안내 성공 | 관리자 권한 앱에서 동작하지 않을 수 있음을 확인할 수 있다 |

# 25. PRD 요약

이 제품은 **Windows용 개인 프롬프트 런처**다.

핵심은 다음 다섯 가지다.

1. **텐키 UI**
   - 실제 텐키 레이아웃을 모방한다.
   - 숫자뿐 아니라 `/`, `*`, `-`, `+`, `.`까지 슬롯으로 사용한다.

2. **2단계 선택 구조**
   - 홈 화면에서 카테고리 선택
   - 카테고리 화면에서 문구 선택

3. **LLM 프롬프트에 최적화**
   - 긴 텍스트
   - 줄바꿈
   - Markdown
   - 코드블록
   - 반복 사용 프롬프트 저장

4. **빠른 인지와 실행**
   - 각 슬롯에 이미지 첨부 가능
   - 앱이 자동으로 썸네일 처리
   - 텍스트보다 이미지와 위치 기억으로 빠르게 실행
   - 선택 즉시 붙여넣기
   - 기존 클립보드 복원
   - 붙여넣기 후 기본 자동 숨김

5. **Windows 네이티브 구현**
   - C# / .NET 10 LTS / WPF 기반 구현
   - Win32 API로 전역 단축키와 붙여넣기 처리
   - SQLite와 AppData 폴더로 로컬 데이터 저장
   - 배포, 서버, 계정, 클라우드 없이 개인용으로 사용

이 PRD의 MVP 범위는 **프롬프트 저장, 텐키 선택, 이미지 슬롯, 로컬 데이터 저장, Windows 붙여넣기 경험**에 집중한다.

키 리매핑, 프로그램 실행, 볼륨 조절, 매크로, 배포/설치 프로그램은 MVP 이후 또는 범위 밖으로 둔다.
