# PRD v0.3

# DeckDeckDeck

## 0. 문서 목적

이 문서는 DeckDeckDeck의 현재 완료 상태와 다음 MVP 범위를 정리한다.

기존 PRD v0.2는 첫 MVP를 정의하기 위한 긴 요구사항 문서였고, 현재는 대부분 구현이 완료되었다.  
따라서 v0.3에서는 완료된 기능을 기준 상태로 반영하고, 다음 MVP에서 새로 다룰 기능만 작게 정의한다.

시각 디자인 기준은 계속 [DESIGN_GUIDELINES.md](DESIGN_GUIDELINES.md)를 따른다.

---

## 1. 제품 정의

DeckDeckDeck은 Windows에서 텐키 단축키와 텐키 모양 UI를 사용해 저장된 문구를 빠르게 실행하는 개인용 로컬 데스크톱 앱이다.

현재 핵심 사용 흐름은 다음과 같다.

```text
Ctrl + Numpad 0
→ 홈 화면
→ 덱 선택
→ 문구 선택
→ 현재 커서 위치에 붙여넣기
```

또는:

```text
Ctrl + Numpad 1~9 또는 기호
→ 해당 덱 바로 진입
→ 문구 선택
→ 현재 커서 위치에 붙여넣기
```

| 실제 붙여넣기 안정성 | 사용자가 여러 앱에서 반복 사용하며 안정적으로 작동함을 확인 |

## 4. 다음 MVP 목표

다음 MVP의 목표는 **문구 붙여넣기 앱에서 작은 실행 런처로 확장**하는 것이다.

현재 카드는 문구 붙여넣기만 실행한다.  
다음 MVP에서는 카드가 아래 세 종류의 실행 결과를 가질 수 있게 한다.

1. 문구 붙여넣기
2. 프로그램 또는 파일 실행
3. 웹페이지 주소 열기

또한 실제 사용 중 편집 속도를 높이기 위해, 카드 우클릭으로 바로 편집하는 빠른 편집 모드를 추가한다.

## 5. 다음 MVP 포함 범위

### 5.1 DB 오류 대응

앱 시작 또는 사용 중 DB 접근에 문제가 생겨도 앱이 무한 크래시하지 않게 한다.

| ID         | 요구사항                                                 |
| ---------- | ---------------------------------------------------- |
| NFR-DB-001 | DB 파일을 열 수 없을 때 앱이 바로 종료되지 않아야 한다                    |
| NFR-DB-002 | DB 오류가 발생하면 사용자에게 짧은 안내를 보여준다                        |
| NFR-DB-003 | 오류 내용은 로컬 로그에 남긴다                                    |
| NFR-DB-004 | DB 오류 상태에서는 가능한 범위에서 설정 화면 또는 안내 화면을 열 수 있어야 한다      |
| NFR-DB-005 | 자동 복구나 DB 재생성은 사용자의 기존 데이터를 지울 수 있으므로 확인 없이 실행하지 않는다 |

성공 기준:

- DB 파일이 잠겨 있거나 손상된 상황에서도 앱이 반복 크래시하지 않는다.
- 사용자는 “DB 문제로 현재 데이터를 불러올 수 없다”는 상황을 이해할 수 있다.
- 로그 파일에서 오류 원인을 확인할 수 있다.

### 5.2 프로그램/파일 실행 카드

카드 실행 결과를 문구 붙여넣기뿐 아니라 앱 또는 파일 실행으로 확장한다.
이번 MVP에서는 `.exe`, 일반 파일, 폴더를 Windows 기본 방식으로 여는 것까지만 포함한다.
명령어 실행, 인자 전달, 작업 폴더 지정, 관리자 권한 실행은 추후 실행 타입 확장 후보로 둔다.

| ID         | 요구사항                                                 |
| ---------- | ---------------------------------------------------- |
| FR-ACT-001 | 문구 카드는 실행 타입을 가진다                                    |
| FR-ACT-002 | 실행 타입은 우선 `문구 붙여넣기`와 `프로그램/파일 실행` 두 가지를 지원한다         |
| FR-ACT-003 | 프로그램/파일 실행 카드는 실행할 파일 경로를 저장한다                       |
| FR-ACT-004 | 사용자는 문구 편집 화면에서 버튼 토글로 실행 타입을 선택할 수 있다               |
| FR-ACT-005 | 실행 타입이 프로그램/파일 실행이면 문구 내용 입력 대신 파일 또는 폴더 선택을 중심으로 보여준다 |
| FR-ACT-006 | 카드 선택 시 파일 경로를 Windows 기본 방식으로 실행한다                  |
| FR-ACT-007 | 실행 파일이나 대상 파일이 없으면 앱이 종료되지 않고 오류 메시지 또는 상태 메시지를 보여준다 |
| FR-ACT-008 | 프로그램/파일 실행 후 자동 숨김 설정을 적용한다                          |

일반 파일 범위:

- 일반 파일은 Windows가 기본 앱 연결 또는 기본 실행 방식으로 처리할 수 있는 파일을 의미한다.
- `.txt`, `.pdf`, `.docx` 같은 문서 파일은 연결된 기본 앱으로 열린다.
- `.bat`, `.cmd`, `.ps1` 같은 스크립트 파일도 경로 대상으로 저장할 수 있다.
- 단, 스크립트 파일의 실제 동작은 Windows 파일 연결, 실행 정책, 보안 설정을 따른다.
- 앱은 스크립트 실행 정책을 우회하거나 관리자 권한으로 강제 실행하지 않는다.

구현 메모:

- `LaunchFile`은 현재는 파일/폴더 경로 실행만 의미한다.
- 추후 명령어 실행이 필요해지면 `LaunchCommand` 또는 별도 실행 타입을 추가해 `LaunchFile`과 분리한다.
- `LaunchFile`에 명령어 문자열, 인자, 파이프, 쉘 문법을 섞어 넣지 않는다.

성공 기준:

- 카드 하나를 `.exe`, 문서, 폴더 중 하나에 연결할 수 있다.
- 해당 카드를 누르면 Windows에서 대상이 실행된다.
- 대상 파일이 없거나 실행에 실패해도 앱이 죽지 않는다.

### 5.3 빠른 편집 모드

카드 우클릭으로 해당 카드를 바로 수정할 수 있게 한다.

| ID           | 요구사항                                       |
| ------------ | ------------------------------------------ |
| FR-QEDIT-001 | 홈 화면에서 덱 카드를 우클릭하면 덱 편집 화면을 연다             |
| FR-QEDIT-002 | 덱 화면에서 문구 카드를 우클릭하면 문구 편집 화면을 연다           |
| FR-QEDIT-003 | 빈 카드 우클릭도 기존 톱니모양 아이콘과 동일하게 새 항목 편집 화면을 연다 |
| FR-QEDIT-004 | 비활성 카드 우클릭도 다시 활성화할 수 있도록 편집 화면을 연다        |
| FR-QEDIT-005 | 기존 톱니모양 아이콘은 유지한다                          |

성공 기준:

- 카드 본문 좌클릭은 기존 실행 동작을 유지한다.
- 카드 우클릭은 편집 진입으로 동작한다.
- 톱니모양 아이콘 없이도 빠르게 수정할 수 있다.

### 5.4 웹페이지 주소 열기 카드

카드 실행 결과를 웹페이지 주소 열기로도 확장한다.
이번 MVP에서는 사용자가 저장한 웹페이지 주소를 Windows 기본 브라우저로 여는 것까지만 포함한다.
앱 내부 브라우저, 브라우저 선택, 새 창/새 탭 제어, 로그인 상태 관리는 포함하지 않는다.

| ID         | 요구사항                                                 |
| ---------- | ---------------------------------------------------- |
| FR-WEB-001 | 실행 타입은 `웹페이지 주소 열기`를 추가로 지원한다                    |
| FR-WEB-002 | 웹페이지 주소 열기 카드는 열 웹페이지 주소를 저장한다                    |
| FR-WEB-003 | 사용자는 문구 편집 화면에서 실행 타입을 웹페이지 주소 열기로 선택할 수 있다      |
| FR-WEB-004 | 실행 타입이 웹페이지 주소 열기이면 문구 내용 입력 대신 웹페이지 주소 입력을 중심으로 보여준다 |
| FR-WEB-005 | 카드 선택 시 저장된 주소를 Windows 기본 브라우저로 연다               |
| FR-WEB-006 | 주소가 비어 있거나 열 수 없는 형식이면 앱이 종료되지 않고 오류 메시지 또는 상태 메시지를 보여준다 |
| FR-WEB-007 | 웹페이지 주소 열기 후 자동 숨김 설정을 적용한다                       |

웹페이지 주소 범위:

- `https://` 또는 `http://` 주소를 기본 대상으로 한다.
- 사용자가 입력한 주소에 프로토콜이 없으면 `https://`를 붙여 여는 방식을 검토한다.
- 검색어 실행, 브라우저별 프로필 선택, 시크릿 창 실행은 이번 MVP에서 제외한다.

구현 메모:

- `LaunchUrl`은 웹페이지 주소 실행만 의미한다.
- `LaunchUrl`에 명령어 문자열, 파일 경로, 쉘 문법을 섞어 넣지 않는다.
- 실행은 Windows 기본 브라우저 연결을 따른다.

성공 기준:

- 카드 하나를 `https://` 주소에 연결할 수 있다.
- 해당 카드를 누르면 기본 브라우저에서 웹페이지가 열린다.
- 주소가 비어 있거나 잘못되어도 앱이 죽지 않는다.

## 7. 데이터 구조 변경 방향

다음 MVP에서는 기존 `Snippet`이 문구만 의미하는 이름에서 벗어날 수 있다.  
다만 큰 구조 변경은 피하고, 우선 기존 테이블에 실행 타입 필드를 추가하는 작은 변경을 검토한다.

초안:

| 필드 | 설명 |
|---|---|
| `ActionType` | `PasteText`, `LaunchFile`, `LaunchUrl` |
| `Content` | `PasteText`일 때 붙여넣을 문구 |
| `LaunchPath` | `LaunchFile`일 때 실행할 파일/폴더 경로 |
| `LaunchUrl` | `LaunchUrl`일 때 열 웹페이지 주소 |

주의:

- 기존 문구 데이터는 모두 `PasteText`로 유지되어야 한다.
- `LaunchFile`에서는 `Content`를 실행에 사용하지 않는다. 기존 테이블 호환을 위해 빈 문자열로 저장할 수 있다.
- `PasteText`에서는 `Content`가 필수이고, `LaunchPath`는 비어 있을 수 있다.
- `LaunchFile`에서는 `LaunchPath`가 필수이고, `Content`는 비어 있을 수 있다.
- `LaunchUrl`에서는 `LaunchUrl`이 필수이고, `Content`와 `LaunchPath`는 비어 있을 수 있다.
- 마이그레이션 또는 DB 보정 중 기존 데이터가 삭제되면 안 된다.
- 실행 타입 확장은 문구 붙여넣기 안정성을 해치지 않아야 한다.

## 8. 다음 MVP 우선순위

1. DB 오류 대응
   - 확인 방법: DB 파일 잠금/잘못된 DB 경로 상황에서도 앱이 반복 크래시하지 않는지 확인

2. 실행 타입 데이터 모델 추가
   - 확인 방법: 기존 문구 데이터가 유지되고 새 실행 타입을 저장할 수 있는지 확인

3. 프로그램/파일 실행 카드 구현
   - 확인 방법: `.exe`, 폴더, 일반 문서 실행을 각각 확인

4. 웹페이지 주소 열기 카드 구현
   - 확인 방법: `https://` 주소가 기본 브라우저로 열리는지 확인

5. 빠른 편집 모드 구현
   - 확인 방법: 홈/덱 화면에서 우클릭 편집 진입 확인

6. 회귀 테스트
   - 확인 방법: 기존 문구 붙여넣기, 클립보드 복원, 자동 숨김이 그대로 작동하는지 확인

## 9. 검증 체크리스트

다음 MVP 완료 시 아래를 확인한다.

- `dotnet test`가 통과한다.
- 기존 문구 카드가 그대로 붙여넣기 된다.
- 기존 이미지와 썸네일이 그대로 표시된다.
- 새 프로그램/파일 실행 카드가 실행된다.
- 새 웹페이지 주소 열기 카드가 기본 브라우저로 열린다.
- 실행 대상이 없을 때 앱이 죽지 않는다.
- 웹페이지 주소가 비어 있거나 잘못되었을 때 앱이 죽지 않는다.
- 카드 우클릭으로 편집 화면에 들어간다.
- 비활성 카드도 우클릭으로 편집 화면에 들어간다.
- DB 오류 상황에서 앱이 반복 크래시하지 않는다.
- 테스트 통과 후 최신 `DeckDeckDeck.exe`를 레포 루트에 준비한다.

## 10. 용어

| 용어 | 뜻 |
|---|---|
| 홈 화면 | 덱을 고르는 첫 화면 |
| 덱 화면 | 특정 덱 안의 카드를 고르는 화면 |
| 카드 | 텐키 한 키에 대응되는 실행 단위 |
| 문구 카드 | 선택하면 저장된 텍스트를 붙여넣는 카드 |
| 실행 카드 | 선택하면 프로그램, 파일, 폴더를 여는 카드 |
| 슬롯 | 카드가 놓이는 텐키 위치 |
| 빠른 편집 | 카드 우클릭으로 바로 편집하는 동작 |
| 붙여넣기 모드 | 기존 입력창 위치를 유지한 채 카드를 고르는 상태 |
| 편집 모드 | DeckDeckDeck 창이 포커스를 가져와 내용을 수정하는 상태 |
